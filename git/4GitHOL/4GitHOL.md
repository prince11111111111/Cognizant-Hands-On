# 4GitHOL: Resolving Merge Conflicts in Git

## Objectives
- Understand why merge conflicts happen during concurrent updates to master and feature branches.
- Implement conflict resolution when modifying identical file lines in master and `GitWork` branch.
- Inspect conflict markers (`<<<<<<<`, `=======`, `>>>>>>>`), resolve via 3-way merge tools (P4Merge / manual), commit resolution, and update `.gitignore` for merge backup files.

---

## Step-by-Step Hands-On Instructions & Execution Log

### Step 1: Verify Master is in Clean State

```bash
git status
```
*Expected Output:*
```text
On branch master
nothing to commit, working tree clean
```

---

### Step 2 & 3: Create Branch `GitWork` and Add `hello.xml`

```bash
git checkout -b GitWork
```
*Output:* `Switched to a new branch 'GitWork'`

Create `hello.xml` with content for `GitWork` branch:
```bash
echo "<greeting><message>Hello from GitWork Feature Branch</message></greeting>" > hello.xml
```

Check status:
```bash
git status
```
*Output:* Shows `hello.xml` as an untracked file on branch `GitWork`.

---

### Step 4: Commit Changes to `GitWork` Branch

```bash
git add hello.xml
git commit -m "Add hello.xml on GitWork branch"
```

---

### Step 5 & 6: Switch to Master and Add Conflicting `hello.xml`

```bash
git checkout master
```
*Output:* `Switched to branch 'master'`

Create `hello.xml` on `master` with **different content**:
```bash
echo "<greeting><message>Hello from Master Trunk Branch</message></greeting>" > hello.xml
```

---

### Step 7: Commit Conflicting Changes to Master

```bash
git add hello.xml
git commit -m "Add hello.xml on master branch with master content"
```

---

### Step 8 & 9: Observe Log Graph and CLI Differences

1. **Observe log graph across all branches:**
   ```bash
   git log --oneline --graph --decorate --all
   ```
   *Output:*
   ```text
   * c7d8e9f (HEAD -> master) Add hello.xml on master branch with master content
   | * a1b2c3d (GitWork) Add hello.xml on GitWork branch
   |/
   * 5d6e7f8 Initial commit on master trunk
   ```

2. **Check CLI differences between master and GitWork:**
   ```bash
   git diff master GitWork
   ```
   *Output:*
   ```diff
   --- a/hello.xml
   +++ b/hello.xml
   @@ -1 +1 @@
   -<greeting><message>Hello from Master Trunk Branch</message></greeting>
   +<greeting><message>Hello from GitWork Feature Branch</message></greeting>
   ```

---

### Step 10: Visual Differences via P4Merge Tool

```bash
git difftool master GitWork
```

---

### Step 11 & 12: Merge Branch `GitWork` to Master and Observe Conflict Markup

Execute merge:
```bash
git merge GitWork
```
*Output:*
```text
Auto-merging hello.xml
CONFLICT (add/add): Merge conflict in hello.xml
Automatic merge failed; fix conflicts and then commit the result.
```

Inspect `hello.xml` to observe Git conflict markup:
```bash
cat hello.xml
```
*Content:*
```xml
<<<<<<< HEAD
<greeting><message>Hello from Master Trunk Branch</message></greeting>
=======
<greeting><message>Hello from GitWork Feature Branch</message></greeting>
>>>>>>> GitWork
```

---

### Step 13 & 14: Resolve Conflict using 3-Way Merge Tool / Manual Edit & Commit

1. **Open 3-way merge tool or edit `hello.xml`:**
   ```bash
   git mergetool
   ```
   *Or manually edit `hello.xml` to combine the changes:*
   ```xml
   <greeting>
       <message>Hello from Master Trunk Branch</message>
       <message>Hello from GitWork Feature Branch</message>
   </greeting>
   ```

2. **Stage and commit resolved conflict:**
   ```bash
   git add hello.xml
   git commit -m "Merge branch 'GitWork' into master and resolve conflict in hello.xml"
   ```

---

### Step 15 & 16: Update `.gitignore` for Merge Backup Files (`*.orig`)

1. **Check status to observe backup files generated by mergetool:**
   ```bash
   git status
   ```
   *Output shows untracked file `hello.xml.orig`.*

2. **Add backup file rule `*.orig` to `.gitignore`:**
   ```bash
   echo "*.orig" >> .gitignore
   ```

3. **Stage and commit `.gitignore` updates:**
   ```bash
   git add .gitignore
   git commit -m "Add *.orig mergetool backup files to .gitignore"
   ```

---

### Step 17, 18 & 19: List Branches, Delete Merged Branch, and Inspect Final Log

1. **List all available branches:**
   ```bash
   git branch -a
   ```
   *Output:*
   ```text
   * master
     GitWork
   ```

2. **Delete the merged branch `GitWork`:**
   ```bash
   git branch -d GitWork
   ```
   *Output:* `Deleted branch GitWork (was a1b2c3d).`

3. **Observe final log decorated graph:**
   ```bash
   git log --oneline --graph --decorate
   ```
   *Output Example:*
   ```text
   * 8f9e0d1 (HEAD -> master) Add *.orig mergetool backup files to .gitignore
   * e4f5a6b Merge branch 'GitWork' into master and resolve conflict in hello.xml
   |\  
   | * a1b2c3d Add hello.xml on GitWork branch
   * | c7d8e9f Add hello.xml on master branch with master content
   |/  
   * 5d6e7f8 Initial commit on master trunk
   ```
